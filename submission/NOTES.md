# NOTES — best configuration and why it works

My best model is a 1.92M-parameter decoder-only GPT (n_embd 192, 3 layers, 4 heads, RoPE positions, block 256, tied input/output embeddings) trained for 2,000 Adam steps at lr 1e-3 (cosine-decayed to 1e-4 after 100 warmup steps, init std 0.05) on a byte-level BPE tokenizer of vocab 3,072 trained only on the corpus, and it scores **1.7416 dev bpb** versus the baseline's 2.3718 (−26.6%). The single biggest win is the tokenizer: the corpus is ~33% Devanagari *by bytes* (each Hindi code point is 3 UTF-8 bytes), so the byte tokenizer wastes 3 prediction slots per Hindi character, whereas BPE compressing to 3.71 bytes/token lets 2,000 steps see roughly a full epoch instead of under 30% of the data. Weight tying and RoPE are what make that vocab affordable — tying removes a separate vocab×d output matrix and RoPE removes the learned position table — so every spare parameter goes into vocabulary. Within the 2M cap, spending parameters on compression beat spending them on depth or width (vocab 3072/L3 > 2048/L3 > 1024/L4). The trainer recipe matters almost as much as size: raising LR 3e-4→1e-3 and adding warmup + cosine decay were the biggest non-tokenizer gains. Counter-intuitively, textbook "improvements" *hurt* in this <1-epoch regime — AdamW with weight decay 0.1 and the standard 0.02 init both regressed, because the model is underfitting (not overfitting), so decay and small init only slow the little learning 2,000 steps buy; I therefore kept plain Adam, no weight decay, and a larger 0.05 init, with GPT-2 residual-projection scaling and grad-clipping as cheap insurance. The ambitious swap I tried, the Lion optimizer, lost to Adam at the same learning rate because its sign-based updates need a much smaller LR — diagnosed and documented in RUNLOG. Everything obeys the caps (2,000 steps, <2M params, corpus-only, pure PyTorch/stdlib, CPU) and the tokenizer is lossless with a full byte fallback, so `python evaluate.py --checkpoint ckpt.pt --text_file <file>` runs unmodified on any UTF-8 text.
